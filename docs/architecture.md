# Архитектура OPFS Studio

Приложение построено на трёх слоях:

### 1. Панель (UI)

- Интерфейс на React.
- Отображает файловое дерево и редактор.
- Отправляет команды (создание, запись, переименование) через сообщения.

### 2. Content-script

- Получает команды от панели.
- Выполняет работу с OPFS API.
- Следит за изменениями файлов и отправляет уведомления обратно в UI.

### 3. OPFS Watcher

- Обёртка над `navigator.storage.getDirectory()`.
- Реализует операции: `readFile`, `writeFile`, `rename`, `mkdir`, `delete`.
- Следит за изменениями и сравнивает их с локальными версиями.

---

### Поток данных

1. Пользователь инициирует действие в UI (например, создаёт файл).
2. Панель шлёт сообщение в content-script.
3. Content-script вызывает функции OPFS Watcher.
4. После завершения операции отправляется обратное сообщение в UI.
5. UI обновляет состояние.

---

### Обработка изменений снаружи

Watcher сравнивает `lastModified` и `size` файлов.  
Если файл изменён вне редактора → UI показывает баннер конфликта.
